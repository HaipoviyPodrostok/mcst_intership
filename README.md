# Графовые Алгоритмы

## Реализованные алгоритмы
1. **RPO (Reverse Post-Order)** — обход графа в глубину (DFS) с нумерацией узлов в обратном порядке завершения обработки.
2. **Алгоритм Дейкстры** — поиск кратчайших путей от заданного стартового узла до всех остальных.
3. **Алгоритм Форда-Фалкерсона** — поиск максимального потока в сети между источником и стоком. Вес ребра используется как пропускная способность.
4. **Алгоритм Тарьяна** — поиск сильно связных компонент (SCC) графа.

## Требования
- **C++23**
- CMake 3.14 или выше
- Graphiz (для дампов)
- googletest (опционально)
- python (для e2e тестов)
## Сборка проекта

```bash
cmake -S . -B build
cmake --build build -j$(nproc) 
```

В директории `build` появится исполняемый файл `a.out`.

### ENABLE_DUMP
Если при конфигурации CMake передать флаг `-DENABLE_DUMP=ON`, появитсякоманда `DUMP`, которая генерирует файлы графов в формате `.dot` для Graphviz в папке `dumps/dots/`.

```bash
cmake .. -DENABLE_DUMP=ON
make -j$(nproc)
```

### Сборка и запуск тестов
Проект содержит unit-тесты на базе Google Test и E2E тесты

Чтобы собрать проект с тестами, включите флаг `-DBUILD_TESTING=ON`:

```bash
cmake -S . -B build -DBUILD_TESTING=ON
cmake --build build -j$(nproc) 
```

Запустить все тесты можно с помощью CTest из папки `build`:
```bash
ctest --output-on-failure
```

## Синтаксис команд

| Команда | Описание | Пример вывода |
| --- | --- | --- |
| `NODE <Имя>` | Добавляет узел в граф. | (нет вывода) |
| `EDGE <От> <Куда> <Вес>` | Добавляет направленное ребро. Если узлов нет, выведет сообщение об ошибке. | `Unknown node A` |
| `REMOVE NODE <Имя>` | Удаляет узел и все связанные с ним рёбра. | (нет вывода) |
| `REMOVE EDGE <От> <Куда>` | Удаляет ребро между узлами. | (нет вывода) |
| `RPO_NUMBERING <Узел>` | Запускает DFS от указанного узла. Если найдены циклы, сообщает об этом. Затем выводит узлы в порядке RPO. | `Found loop A->B`<br>`A C B D` |
| `DIJKSTRA <Узел>` | Запускает алгоритм Дейкстры и построчно выводит расстояния до всех достижимых узлов. | `B 10`<br>`C 20` |
| `MAXFLOW <Исток> <Сток>` | Вычисляет и выводит максимальный поток. | `25` |
| `TARJAN <Узел>` | Ищет сильно связные компоненты. Выводит только те, чей размер > 1. | `A B C`<br>`D E` |

## Пример использования
